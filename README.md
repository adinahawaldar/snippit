# SNIPPIT.  (Open Source SaaS Starter Registry)

This project is an open-source SaaS starter registry released under the MIT License.  
It is designed to provide developers with production-ready landing pages, UI snippets, AI toolkit components, background designs, and curated open-source resources.

The repository is maintained as a collaborative ecosystem where developers can contribute high-quality assets and help accelerate the development workflow for the global developer community.

## Purpose

The objective of this repository is to standardize and centralize reusable SaaS resources so developers can rapidly integrate high-quality UI sections and tools into their projects.  
All contributions should maintain consistency, scalability, and production-grade quality.

---

## Contribution Scope

Contributors may submit additions or improvements in the following areas:

- SaaS Landing Page Templates
- AI Toolkit Components and Widgets
- UI Snippets and Reusable Patterns
- Background Designs (Gradients, Mesh, Visual Systems)
- Open Source Repository Showcases
- Publications, Case Studies, and Technical Resources

## Standard Contribution Workflow

Please follow the complete workflow below to ensure a consistent and maintainable contribution process.

### 1. Fork the Repository
Create a personal fork of the repository to your GitHub account.

### 2. Clone Your Fork Locally
```bash
git clone https://github.com/adinahawaldar/snippit
cd snippit
```


3. Create a Feature (Head) Branch

All changes must be made in a dedicated feature branch, not directly in the main branch.

```bash

git checkout -b feat/your-contribution-name
```



4. Add Your Contribution in the Correct Directory

Use the following directory mapping to place your contribution appropriately:

## Contribution Directory Mapping

| Contribution Type        | Directory Path                 |
|--------------------------|--------------------------------|
| Landing Page Templates   | `src/pages/LandingPage.jsx/`      |
| AI Toolkit Components    | `src/components/AIToolkitDiscovery.jsx/`    |
| Publications / Articles  | `src/components/ReserchPublication/`      |
| Open Source Showcases    | `src/pages/ProjectRegistry.jsx/`        |
| UI Snippets              | `src/pages/UIComponents.jsx/`        |
| Background Designs       | `src/pages/BackgroundFX.jsx/`       |



5. Commit Changes with Clear Messages

```bash
git add .
git commit -m "feat: add new landing page template"
```

### Push to Your Feature Branch

After committing your changes, push the same feature branch you created:

```bash
git push origin <feature-branch-name>
```
### 7. Open a Pull Request

After pushing your feature branch, open a Pull Request (PR) to merge your changes into the `main` branch of the original repository.

Steps:
1. Navigate to your forked repository on GitHub
2. Click on “Compare & pull request”
3. Provide a clear and concise description of your contribution
4. Mention the exact directory path you modified (e.g., `src/pages/LandingPages/`)
5. Attach screenshots if the contribution is UI-related

### 8. Review and Merge Process

All pull requests will be reviewed for:
- Code quality and readability
- Correct directory placement
- Reusability and modular structure
- Visual and functional consistency with the project

Maintainers may request changes if required.  
Once approved, your contribution will be merged into the `main` branch.
